# インタラクティブイベント支援システム  
（クイズ・参加型コンテンツ・スタンプラリー）

## 1. システム概要

本システムは、リアル会場で実施されるイベント（例：地方移住イベント、パネルディスカッション等）において、  
参加者がスマートフォンを用いてクイズや投票、スタンプラリーなどの **参加型コンテンツ** に参加できるようにするための Web アプリケーションである。

- **目的**：会場での体験価値を高め、参加者同士や地域とのつながりを生み出す
- **構成**：
  - フロントエンド：参加者・管理者それぞれの Web 画面
  - バックエンド：イベント情報・クイズ・投票・スタンプ等を管理する API サーバー
- **対象外**：集客・募集・予約機能  
  （※ 当日会場での体験支援に特化）

---

## 2. ユーザー種別

### 1. 参加者（Participant）

- 会場に来場し、スマートフォン等からシステムにアクセスするユーザー  
- イベントコードやQRコードを用いてコンテンツに参加する

### 2. 管理者 / スタッフ（Admin）

- イベント運営側の担当者  
- イベント・セッション・クイズ・スタンプスポット等を事前に登録・当日運用する

---

## 3. 想定ユースケース

### 3.1 参加者側

- イベントコードまたはQRコードでイベントに参加
- クイズ大会や投票にリアルタイムで参加
- 会場内のブースを回ってスタンプを集める
- 自分やチームのスコア・ランキングを確認する

### 3.2 管理者側

- イベント・セッション・クイズ・投票・スタンプを事前登録
- 当日の進行に合わせて「受付中コンテンツ」を切り替え
- 回答・投票状況をリアルタイムモニタリング
- イベント終了後に結果を集計・エクスポート

---

## 4. 画面・機能一覧

### 4.1 参加者向け機能

#### 1. イベント参加画面
- イベントコード入力またはQRコードで入室
- ニックネーム（表示名）の登録
- チーム名入力／チーム選択

#### 2. 参加メニュー画面
- 現在参加可能なコンテンツ一覧を表示  
  - クイズ  
  - 投票・アンケート  
  - スタンプラリー
- 各コンテンツのステータス表示（受付中／終了／未開始）

#### 3. クイズ参加画面
- 問題文・選択肢の表示（単一選択／複数選択／自由記述対応）
- 回答送信、再回答可否（設定による）
